分享和导出是把分析结果交付给他人的两种方式。分享用于在线查看，导出用于离线留档、会议汇报、邮件分发或二次处理。正式环境中，分享和导出都要注意数据范围和访问权限。

<Callout type="info" title="示例说明">
  本章以“把一个已发布仪表盘临时分享给项目组”和“导出会议材料”为例说明操作方式。示例只是流程说明，实际有效期、密码、字段范围和导出格式应按企业数据安全要求配置。
</Callout>

## 分享和导出的区别

| 方式 | 适合场景 | 风险点 |
| --- | --- | --- |
| 分享 | 在线查看仪表盘或大屏 | 链接被转发、过期未关闭 |
| 导出 | 下载图片、PDF、Excel 等文件 | 文件离开系统后难以追踪 |

如果数据包含客户、人员、财务、合同、成本、利润等敏感信息，优先使用登录访问和资源权限，不建议长期公开分享。

## 分享适用场景

| 场景 | 建议 |
| --- | --- |
| 临时给同事查看 | 设置短有效期，必要时设置密码 |
| 项目验收 | 验收结束后关闭分享链接 |
| 会议室展示 | 使用受控链接或展示账号 |
| 给外部合作方查看 | 只展示脱敏汇总数据，设置密码和有效期 |
| 长期公开展示 | 明确负责人，定期复核链接 |

## 开启分享

不同资源的分享入口可能在资源列表、资源操作菜单或预览页面中。一般流程如下：

![仪表盘列表](/images/docs/crest/screenshots/panel-list.png)

仪表盘列表是常见的分享起点。先在列表中确认资源名称、目录和更新时间，再打开预览或操作菜单。正式环境不应在“测试看板”“临时页面”这类资源上直接生成对外链接，避免未确认数据对外传播。

![数据大屏列表](/images/docs/crest/screenshots/screen-list.png)

数据大屏也可以作为分享对象。大屏分享前要额外检查画布比例、字体、背景图片和预览效果，因为外部访问者通常只看到最终画面，无法进入编辑器协助发现组件遮挡或图表空白。

<Steps>
  <Step>
    ### 找到资源
    进入仪表盘或数据大屏列表，找到要分享的资源。
  </Step>
  <Step>
    ### 打开分享设置
    在资源操作中选择分享或公开链接相关入口。
  </Step>
  <Step>
    ### 开启分享
    打开分享开关。若系统管理员禁用了分享，则无法开启。
  </Step>
  <Step>
    ### 设置有效期
    按业务需要设置到期时间。临时协作不要使用长期有效链接。
  </Step>
  <Step>
    ### 设置访问密码
    外部访问建议启用访问密码。
  </Step>
  <Step>
    ### 复制链接并验证
    用无登录浏览器打开链接，确认访问范围和显示效果。
  </Step>
</Steps>

### 示例：临时分享一个项目看板

假设项目验收需要让外部合作方查看一张只含汇总指标的仪表盘，可以这样配置：

| 配置项 | 示例设置 | 原因 |
| --- | --- | --- |
| 分享对象 | 已发布的项目验收看板 | 草稿页不要分享 |
| 有效期 | 7 天 | 验收结束后自动失效 |
| 访问密码 | 开启 | 防止链接被转发后直接访问 |
| 默认筛选 | 固定到目标项目 | 避免外部人员看到其他项目 |
| 明细字段 | 去掉人员、合同、成本等敏感列 | 对外只保留必要汇总 |

操作顺序：

1. 进入目标仪表盘或大屏的预览页，确认页面已发布。
2. 设置默认查询条件，例如项目、日期范围、组织范围。
3. 从资源操作中打开分享设置。
4. 开启分享，设置有效期和访问密码。
5. 复制链接后，用无登录浏览器打开一次。
6. 输入访问密码，检查页面是否只能看到预期范围。
7. 把链接和密码分开发送，不要写在同一条公开消息里。

分享完成后，建议在系统管理中回到分享管理页面复核一次。公开文档中不建议把分享链接写成长期有效；长期展示应由管理员统一分配访问账号或受控展示地址。

v1.5.7 起，已登录后台的用户访问 `/link/:uuid` 公开分享链接时，也会进入分享展示页，不会被重定向到工作台。公开分享、预览等只读路由按只读访问策略处理；工作台、数据源、数据集等后台路由仍按菜单权限校验。

验证公开分享时，建议分别使用三种状态打开同一链接：

| 访问状态 | 期望结果 |
| --- | --- |
| 未登录浏览器 | 进入分享页，按密码和有效期校验 |
| 已登录普通用户 | 进入分享页，不要求具备后台菜单权限 |
| 已登录管理员 | 仍进入分享页，不自动跳转工作台 |

## 分享管理

管理员可在系统管理中统一查看分享资源。

![分享管理](/images/docs/crest/screenshots/share-management.png)

这张图是管理员治理分享链接的入口。普通用户关注自己是否具备分享权限，管理员需要关注全局：谁创建了链接、链接是否长期有效、资源是否仍在使用、是否包含敏感数据、项目结束后是否关闭。

分享列表通常包含：

| 字段 | 用途 |
| --- | --- |
| 资源名称 | 判断分享的是哪个仪表盘或大屏 |
| 资源类型 | 仪表盘、数据大屏等 |
| 创建人 | 判断链接负责人 |
| 创建时间 | 判断链接是否过久 |
| 有效期 | 判断是否即将过期或长期有效 |
| 状态 | 当前是否可访问 |
| 操作 | 复制、修改、关闭或删除 |

## 分享前检查

1. 页面是否已发布。
2. 页面中是否包含敏感字段。
3. 查询默认值是否正确。
4. 外部访问是否需要密码。
5. 有效期是否明确。
6. 页面中的跳转链接是否会暴露更多资源。
7. 使用无登录浏览器访问是否符合预期。

![数据大屏预览](/images/docs/crest/screenshots/screen-preview.png)

分享前建议先打开预览页。预览页最接近访问者看到的状态，可以检查标题、图表、筛选默认值、字体、图片、颜色和数据刷新时间。对外分享的大屏尤其要确认边缘留白和全屏展示效果。

## 关闭分享

以下情况应及时关闭分享：

- 项目验收结束。
- 页面已下线或数据不再更新。
- 分享人离职或转岗。
- 链接被转发到不应访问的人群。
- 页面包含敏感指标。
- 外部合作结束。

关闭分享后，已复制的链接应无法继续访问。

关闭分享时不能只删除聊天记录里的链接。应回到 Crest 的分享管理或资源分享设置中关闭链接，并用无登录浏览器重新打开原链接，确认已经无法访问。

## 导出中心

导出任务统一进入导出中心。用户可查看任务状态、下载文件、重试或删除记录。

![导出中心](/images/docs/crest/screenshots/export-center.png)

导出中心不是简单下载列表。它能帮助用户判断导出是否提交成功、是否仍在执行、失败后能否重试、成功文件是否需要及时下载。管理员还可以通过导出中心观察是否存在大量失败任务或异常大文件。

## 常见导出对象

| 对象 | 说明 |
| --- | --- |
| 图表数据 | 导出明细或汇总数据 |
| 仪表盘 | 导出图片或 PDF，视系统能力和配置而定 |
| 数据大屏 | 导出图片、模板或页面资源 |
| 数据集 | 导出预览数据或字段数据 |
| 大屏表格 | 通过导出按钮导出当前筛选结果或全部数据 |

具体导出格式会受资源类型、系统配置和权限影响。

## 数据大屏导出按钮

v1.5.5 起，数据大屏可以在页面中放置“导出按钮”组件，并绑定大屏内的表格视图。用户在门户、预览或发布后的大屏中点击按钮，即可按配置提交 Excel 导出任务。

常见配置：

| 配置 | 说明 |
| --- | --- |
| 当前筛选结果 | 导出时带上查询组件、联动、外部参数、Web 参数和钻取上下文 |
| 全部数据 | 不受当前页面筛选影响，适合受控范围内的完整明细 |
| 透视表带格式导出 | 保留透视表样式和结构，适合会议材料或归档 |
| 自定义按钮文案 | 使用“导出当前明细”“下载项目清单”等业务化文案 |

使用建议：

1. 先确认目标表格字段不包含不应离线传播的敏感信息。
2. 当前筛选结果更适合普通用户，全部数据应由管理员谨慎开放。
3. 导出前在页面上确认筛选条件；导出后到导出中心查看任务状态。
4. 透视表导出后应打开文件检查格式、表头说明和多级表头是否符合交付要求。

## 提交导出

<Steps>
  <Step>
    ### 打开目标资源
    进入图表、仪表盘、数据大屏或数据集页面。
  </Step>
  <Step>
    ### 设置筛选条件
    如果页面有查询组件，先设置需要导出的时间、组织和范围。
  </Step>
  <Step>
    ### 点击导出
    在资源操作区选择导出类型。
  </Step>
  <Step>
    ### 等待任务生成
    导出任务会进入导出中心，状态从等待中变为执行中、成功或失败。
  </Step>
  <Step>
    ### 下载文件
    状态为成功后点击下载，并检查文件内容。
  </Step>
</Steps>

![仪表盘编辑器](/images/docs/crest/screenshots/dashboard-editor.png)

导出前先在页面中确认图表已经加载完成。截图中的仪表盘编辑器适合检查筛选器、图表内容和页面布局；如果图表本身还在加载或已经为空，导出文件通常也不会正确。

![数据集预览](/images/docs/crest/screenshots/dataset-detail-preview.png)

导出明细或汇总数据前，可以回到数据集预览确认源数据是否存在。这样能快速区分“导出失败”和“上游无数据”。如果预览中已经为空，应先排查数据源、SQL、过滤条件、权限和缓存。

### 示例：导出会议用数据

会议导出前，先固定页面状态。筛选条件未确认前不应直接导出。

| 步骤 | 示例操作 | 检查点 |
| --- | --- | --- |
| 选择页面 | 打开“项目交付月报”仪表盘 | 页面已发布，图表加载完成 |
| 设置筛选 | 日期选择本月，项目选择目标项目 | 页面数字和会议口径一致 |
| 导出图表 | 导出趋势图、排行图或明细表 | 文件名能识别页面和日期 |
| 查看任务 | 打开导出中心 | 状态为成功后再下载 |
| 复核文件 | 打开下载文件 | 检查空值、乱码、字段范围 |

如果要导出明细表，先确认是否包含敏感字段。客户名称、手机号、身份证号、合同金额、内部成本、个人绩效等字段，不应随意导出到离线文件。

## 导出状态

| 状态 | 含义 | 操作建议 |
| --- | --- | --- |
| 等待中 | 已提交，尚未开始 | 等待后台任务执行 |
| 执行中 | 文件正在生成 | 不要重复提交过多任务 |
| 成功 | 文件已生成 | 及时下载 |
| 失败 | 导出过程中出错 | 查看错误信息，按排查步骤处理 |
| 已过期 | 文件超过保留时间 | 重新提交导出 |

## 导出失败排查

| 问题 | 可能原因 | 处理 |
| --- | --- | --- |
| 任务一直等待 | 后台任务积压 | 稍后重试或联系管理员查看任务队列 |
| 导出失败 | 数据源连接失败、查询超时 | 检查资源是否能正常打开 |
| 大屏导出按钮无响应 | 目标表格未绑定、表格无数据或当前账号无导出权限 | 回到编辑器检查按钮配置，再用普通账号预览 |
| 大屏导出范围不对 | 选择了全部数据，或筛选条件未生效 | 检查按钮导出范围和页面查询组件 |
| 下载失败 | 文件过期、目录权限、代理超时 | 重新导出或联系管理员 |
| 文件为空 | 筛选条件无数据、权限限制 | 调整筛选条件，用页面预览确认 |
| 文件过大 | 数据范围过宽 | 缩小时间范围或拆分导出 |
| 字体异常 | 导出环境缺少字体 | 在字体管理中补充字体 |

## 导出前建议

1. 先在页面中确认数据正确。
2. 只导出必要范围，不要一次导出过大数据量。
3. 明细导出前确认是否包含敏感字段。
4. 对外发送文件前检查内容和权限。
5. 导出文件不再使用后及时删除或按企业规则归档。

## 分享和导出安全建议

| 场景 | 建议 |
| --- | --- |
| 对外分享 | 使用脱敏数据、密码和有效期 |
| 内部汇报 | 文件只发送给需要查看的人 |
| 长期链接 | 定期复核负责人和有效性 |
| 敏感数据 | 使用登录访问，不使用公开链接 |
| 离职人员 | 检查其创建的分享链接和导出文件 |

## 常见问题

| 问题 | 处理 |
| --- | --- |
| 分享链接打不开 | 检查资源是否发布、分享是否开启、有效期是否过期 |
| 分享页面数据为空 | 检查默认筛选、数据集权限和资源状态 |
| 外部用户无法访问 | 检查域名、HTTPS、防火墙和代理 |
| 导出中心没有任务 | 确认是否点击导出成功，或当前账号是否有任务权限 |
| 文件下载后打不开 | 重新下载，必要时重新导出 |
| 导出内容和页面不一致 | 检查导出前筛选条件和数据刷新时间 |
