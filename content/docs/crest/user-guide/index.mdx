本章面向日常使用 Crest 的业务分析人员、数据开发人员、报表制作人员、部门管理员和项目交付人员。内容按完整操作链路组织，可从登录开始逐步阅读，也可直接进入正在处理的功能页。

进入系统后，建议先从工作台确认自己有哪些入口。工作台会展示资源统计、快速创建、收藏和最近使用，是多数用户每天进入 Crest 后的起点。

![工作台入口](/images/docs/crest/screenshots/workspace.png)

截图中的顶部菜单决定当前账号可进入的模块；左侧卡片可快速查看当前账号可见的仪表盘、数据大屏和数据集数量；中间区域适合从收藏或最近使用继续工作。如果实际菜单和截图不同，通常是权限差异，不能说明系统异常。

```mermaid
flowchart LR

```

## 开始前先确认权限

不同用户看到的菜单不同，这是正常现象。Crest 的菜单、资源和数据范围都受权限控制。

| 目标任务 | 需要确认什么 |
| --- | --- |
| 只查看仪表盘或大屏 | 是否有对应资源查看权限 |
| 新建仪表盘或大屏 | 是否有仪表盘或数据大屏菜单权限、创建权限 |
| 新建数据集 | 是否有数据集菜单权限、数据源访问权限 |
| 新建数据源 | 是否有数据源菜单权限，以及数据库账号信息 |
| 配置分享 | 是否允许开启分享，以及分享策略是否被管理员限制 |
| 导出文件 | 是否有导出权限，数据量是否符合导出限制 |

如看不到手册中的某个入口，不要反复刷新页面。请把“账号、缺少的菜单、资源名称、截图”发给管理员，由管理员检查角色、菜单权限和资源权限。

## 核心概念

| 概念 | 说明 | 操作时怎么理解 |
| --- | --- | --- |
| 数据源 | Crest 读取数据的位置 | 数据库、Excel、API 等连接 |
| 数据集 | 图表使用的数据模型 | 对字段、口径、权限、缓存做统一管理 |
| 维度 | 用来分组、筛选、下钻的字段 | 日期、地区、组织、产品、客户 |
| 指标 | 用来计算和统计的数值字段 | 销售额、订单数、毛利率、工单量 |
| 图表 | 单个可视化组件 | 指标卡、柱状图、折线图、表格 |
| 查询组件 | 页面上的筛选器 | 时间、地区、组织、项目、状态 |
| 仪表盘 | 日常分析页面 | 适合筛选、查看、导出和复盘 |
| 数据大屏 | 全屏展示页面 | 适合会议室、展厅、值班监控 |
| 数据血缘 | 上下游依赖关系 | 删除、改字段、改数据集前先看影响 |

## 推荐学习顺序

<Steps>
  <Step>
    ### 登录与工作台
    先熟悉顶部菜单、账号菜单、快速创建、收藏和最近使用。
  </Step>
  <Step>
    ### 数据源
    学会创建数据库、Excel 或 API 数据源，能测试连接和查看表字段。
  </Step>
  <Step>
    ### 数据集
    学会选择表、写 SQL、配置字段、计算字段、权限和缓存。
  </Step>
  <Step>
    ### 图表
    学会选择图表类型，拖拽维度指标，配置过滤、排序、样式和高级交互。
  </Step>
  <Step>
    ### 仪表盘和数据大屏
    学会把图表、筛选器、文本、图片和其他组件组装成可交付页面。
  </Step>
  <Step>
    ### 分享、导出和血缘
    学会把结果交付出去，并在变更前检查影响范围。
  </Step>
</Steps>

## 一个完整示例

以“月度经营分析仪表盘”为例，推荐按以下顺序操作：

![仪表盘编辑器](/images/docs/crest/screenshots/dashboard-editor.png)

上图是仪表盘编辑器的典型工作界面。左侧用于管理组件和资源，中间是页面画布，右侧用于配置选中组件的数据、样式和交互。制作仪表盘前应先确认数据集字段、图表口径、筛选器和页面布局，再开始放置图表组件。

| 步骤 | 操作 | 结果 |
| --- | --- | --- |
| 1 | 创建“经营数仓-只读”数据源 | Crest 可以读取订单、客户、商品等表 |
| 2 | 创建“月度经营分析数据集” | 字段被整理成日期、区域、产品、销售额、订单数等 |
| 3 | 做指标卡 | 展示销售额、订单数、毛利率 |
| 4 | 做趋势图 | 展示最近 12 个月销售额走势 |
| 5 | 做对比图 | 展示各区域或产品线贡献 |
| 6 | 做明细表 | 展示需要追溯的订单或项目明细 |
| 7 | 创建仪表盘 | 把图表、筛选器和说明文字放到同一页面 |
| 8 | 配置筛选 | 按月份、区域、产品线筛选 |
| 9 | 配置联动 | 点击区域图表后，其他图表同步过滤 |
| 10 | 发布并分享 | 给业务负责人查看，必要时导出文件 |

## 保存、发布、分享的区别

| 操作 | 含义 | 什么时候用 |
| --- | --- | --- |
| 保存 | 保存当前编辑内容 | 编辑过程中随时保存 |
| 发布 | 将当前版本变成可正式查看版本 | 页面确认可交付后 |
| 预览 | 临时查看页面效果 | 发布前检查布局和数据 |
| 分享 | 生成外部访问链接 | 给没有编辑权限的人查看 |
| 导出 | 生成文件 | 汇报、归档或离线处理 |

![分享管理](/images/docs/crest/screenshots/share-management.png)

分享和导出都属于交付动作。分享适合让别人在线查看，导出适合汇报、归档和离线处理。对外分享前应确认数据是否脱敏、链接是否设置有效期、访问人是否符合授权范围。

## 交付前总检查

1. 页面标题是否清楚。
2. 数据源连接是否稳定。
3. 数据集字段口径是否明确。
4. 图表维度和指标是否配置正确。
5. 筛选器是否能影响目标图表。
6. 普通用户权限是否验证过。
7. 分享链接是否设置有效期和密码。
8. 导出文件是否能下载且内容正确。
9. 数据血缘中是否能看到关键依赖。
