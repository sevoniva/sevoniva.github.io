---
title: 权限与账号问题
description: Crest 用户、角色、菜单权限、资源权限和账号生命周期常见问题。
---

权限问题不要只看“用户有没有账号”。Crest 中常见权限由菜单权限、功能权限、资源权限、数据权限共同决定。

![权限管理](/images/docs/crest/screenshots/permission-management.png)

## 用户能登录但看不到菜单

可能原因：

- 用户没有加入任何角色。
- 角色没有配置菜单权限。
- 用户加入了错误组织或角色。
- SSO 自动创建用户后只分配了默认低权限角色。

处理顺序：

1. 进入用户管理，确认用户状态正常。
2. 查看用户所属角色。
3. 进入权限管理，确认角色勾选了对应菜单。
4. 让用户退出后重新登录。
5. 使用普通测试账号复现验证。

## 用户看不到某个仪表盘或大屏

| 可能原因 | 处理 |
| --- | --- |
| 没有资源权限 | 给用户或角色授权该资源 |
| 资源被移动 | 检查资源所在目录或空间 |
| 资源被删除 | 查看审计日志，确认删除记录 |
| 角色权限被回收 | 检查最近权限变更 |
| 分享链接关闭 | 在分享管理中检查链接状态 |

![角色管理](/images/docs/crest/screenshots/role-management.png)

## 用户能看资源但不能编辑

这通常是功能权限或资源操作权限不足。处理方法：

1. 确认该用户是否应该编辑资源。
2. 检查角色是否有编辑功能权限。
3. 检查资源是否授权编辑而不只是查看。
4. 确认资源是否被其他用户锁定或正在编辑。
5. 使用普通测试账号验证按钮是否出现。

## 管理员权限过多怎么办

管理员权限应按职责拆分，不建议所有管理员都拥有完整系统管理权限。

建议拆分：

| 角色 | 权限 |
| --- | --- |
| 平台管理员 | 用户、角色、系统参数、审计 |
| 数据管理员 | 数据源、数据集、缓存同步 |
| 内容管理员 | 仪表盘、大屏、分享和导出 |
| 安全管理员 | 权限复核、审计、SSO |

## 离职账号如何处理

1. 禁用账号。
2. 回收角色。
3. 检查其创建的分享链接。
4. 检查其负责的数据源、数据集、仪表盘和大屏。
5. 将关键资源移交给新负责人。
6. 在审计日志中保留处理记录。

![用户管理](/images/docs/crest/screenshots/user-management.png)

## 权限修改后用户仍然看不到

可能原因：

- 用户未重新登录。
- 浏览器缓存旧菜单。
- 修改的是另一个角色。
- 用户同时属于多个角色，实际权限判断不符合预期。
- 资源权限和菜单权限只配置了其中一项。

处理建议：让用户退出重新登录，并用管理员视角确认用户、角色、菜单和资源授权链路。
