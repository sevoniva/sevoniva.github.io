数据和可视化问题通常要沿着链路排查：数据源 → 数据集 → 图表 → 仪表盘 / 大屏 → 分享 / 导出。不要只看最终页面空白，也要确认上游数据是否可用。

```mermaid
flowchart LR
  A["数据源"] --> B["数据集"]
  B --> C["图表"]
  C --> D["仪表盘 / 数据大屏"]
  D --> E["分享 / 导出"]
```

## 数据源连接失败

![数据源列表](/images/docs/crest/screenshots/datasource-list.png)

| 可能原因 | 处理 |
| --- | --- |
| 数据库地址或端口错误 | 使用服务器网络测试连通性 |
| 用户名或密码错误 | 重新确认数据库账号 |
| 数据库权限不足 | 给账号授予必要库表权限 |
| 网络被防火墙拦截 | 检查安全组、防火墙和网络策略 |
| JDBC 参数不兼容 | 移除危险或不必要参数后重试 |

## 数据集预览为空

![数据集配置](/images/docs/crest/screenshots/dataset-form.png)

排查顺序：

1. 数据源连接是否成功。
2. 表或 SQL 是否有数据。
3. 过滤条件是否过严。
4. 行权限是否限制了当前用户。
5. 字段类型是否识别正确。
6. 缓存是否过期或同步失败。

## 图表为空或指标不正确

![图表编辑器](/images/docs/crest/screenshots/dashboard-editor.png)

常见原因：

- 维度和指标放置不正确。
- 聚合方式不符合业务口径。
- 日期字段格式不正确。
- 过滤条件、联动条件或钻取条件生效。
- 当前用户没有数据权限。
- 数据集字段被删除或重命名。

处理建议：

1. 回到数据集预览确认原始数据。
2. 临时移除图表过滤条件。
3. 检查维度、指标和聚合方式。
4. 使用管理员账号和普通账号分别验证。
5. 查看数据血缘，确认字段来源。

## 仪表盘打开慢

![仪表盘列表](/images/docs/crest/screenshots/panel-list.png)

可能原因：

| 原因 | 处理 |
| --- | --- |
| 页面图表过多 | 拆分仪表盘或减少首屏图表 |
| SQL 复杂 | 优化数据集 SQL 或增加缓存 |
| 数据量过大 | 增加过滤条件、分区或聚合表 |
| 并发高 | 检查数据库连接数和服务资源 |
| 图片或字体过大 | 压缩资源，清理无用字体 |

## 大屏预览样式异常

![数据大屏编辑器](/images/docs/crest/screenshots/screen-editor.png)

处理顺序：

1. 检查浏览器缩放比例。
2. 检查画布尺寸和适配方式。
3. 检查上传图片是否丢失。
4. 检查字体是否在当前环境存在。
5. 检查预览页和编辑器是否同样异常。
6. 对外展示前用目标屏幕分辨率验证。

![数据大屏预览](/images/docs/crest/screenshots/screen-preview.png)

## 导出失败

![导出中心](/images/docs/crest/screenshots/export-center.png)

先在导出中心查看任务状态，再看应用日志。

| 现象 | 处理 |
| --- | --- |
| 一直等待 | 检查后台任务是否积压 |
| 执行后失败 | 查看错误信息和应用日志 |
| 下载失败 | 检查文件是否生成、代理超时和目录权限 |
| 文件内容为空 | 检查资源本身和数据权限 |
| 字体不一致 | 检查字体管理和导出环境 |

## 如何确认字段来源

入口：数据血缘。

![数据血缘](/images/docs/crest/screenshots/lineage-overview.png)

数据血缘适合回答：

- 某个图表字段来自哪个数据集字段。
- 数据集字段来自哪个数据源表字段。
- 修改某个数据源字段会影响哪些图表。
- 删除数据集前会影响哪些仪表盘或大屏。

排查字段问题时，建议先看血缘，再决定是否修改数据源、数据集或图表。
