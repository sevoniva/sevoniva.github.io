本章面向第一次接触 Crest 的用户，帮助你先完成四项基础工作：启动系统、登录后台、认识工作台，并创建第一个可查看的分析页面。

<Callout type="info" title="适合谁阅读">
  如仅用于体验系统，可以按本页操作。正式上线前，请继续阅读安装部署、配置管理、安全说明和管理员指南。
</Callout>

## 准备环境

建议准备一台 Linux 服务器，至少满足以下条件：

| 项目 | 建议配置 |
| --- | --- |
| CPU | 2 核及以上 |
| 内存 | 4 GB 及以上 |
| 磁盘 | 20 GB 可用空间以上 |
| 端口 | 默认开放 `8100/tcp` |
| 运行时 | Docker 与 Docker Compose |

<Callout type="warning" title="生产环境建议">
  正式部署时请使用固定域名和 HTTPS，并把系统访问来源配置为实际域名。不要在生产环境使用公开示例密码。
</Callout>

## 快速安装

进入 Crest 源码中的安装目录执行：

```bash
cd installer
bash install.sh
```

安装脚本会创建运行目录、生成缺省密码和加密参数、启动 MySQL 与 Crest 服务，并在最后输出访问地址、管理员账号和初始密码。

快速体验默认使用 `standard` 加密模式。需要 SM2、SM3、SM4 的环境，请在安装前阅读安装部署章节，设置 `CREST_CRYPTO_MODE=sm-suite` 和 `CREST_SM4_KEY`。启用国密后，本页后续登录、创建数据源、数据集、仪表盘和数据大屏的步骤不变。

默认管理员账号为：

```text
用户名：admin
密码：以安装脚本最终输出为准
```

## 登录系统

打开安装脚本输出的访问地址。默认端口为 `8100`，本机验证时通常是：

```text
http://服务器IP:8100
```

![Crest 登录页](/images/docs/crest/screenshots/login.png)

首次登录后建议立即修改管理员密码。生产环境还应尽早完成 HTTPS、备份、组织、角色和权限配置。

登录后如果页面提示账号或密码错误，请回到安装输出确认初始密码。不要猜测默认密码，也不要在公开环境长期保留初始密码。

## 认识工作台

登录后进入工作台。工作台提供资源概览、快速创建、我的收藏和最近使用。

![Crest 工作台](/images/docs/crest/screenshots/workspace.png)

工作台中的“快速创建”适合用于熟悉产品流程：

1. 点击“数据源”，接入数据库、Excel 或 API。
2. 点击“数据集”，选择表、字段或 SQL 构建分析模型。
3. 点击“仪表盘”或“数据大屏”，把图表组织成展示页面。

## 第一个分析页面

推荐用下面的路径完成第一次体验：

<Steps>
  <Step>
    ### 接入数据源
    在“数据准备 / 数据源”中新建数据源，填写连接信息并测试连接。
  </Step>
  <Step>
    ### 创建数据集
    在“数据准备 / 数据集”中选择数据源表，确认字段后保存数据集。
  </Step>
  <Step>
    ### 制作图表
    在仪表盘或大屏编辑器中添加图表组件，选择数据集，拖拽维度和指标。
  </Step>
  <Step>
    ### 发布页面
    预览确认后保存并发布。需要给外部人员查看时，再开启分享链接。
  </Step>
</Steps>

![数据源列表](/images/docs/crest/screenshots/datasource-list.png)

![数据集列表](/images/docs/crest/screenshots/dataset-list.png)

![仪表盘列表](/images/docs/crest/screenshots/panel-list.png)

如果你的目标是做展示大屏，可以把最后一步换成“新建数据大屏”，在大屏编辑器中添加图表、文字、图片和装饰组件。

![数据大屏列表](/images/docs/crest/screenshots/screen-list.png)

## 快速检查

安装完成后建议检查：

| 检查项 | 期望结果 |
| --- | --- |
| 登录页 | 能正常打开，Logo 和登录框显示完整 |
| 管理员账号 | 使用安装输出的初始密码可以登录 |
| 工作台 | 能看到资源统计和快速创建入口 |
| 系统设置 | 管理员能进入系统参数、用户管理和权限管理 |
| 演示数据 | 新安装环境通常包含演示数据源、数据集和数据大屏 |
