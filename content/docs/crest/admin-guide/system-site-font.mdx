---
title: 系统、站点与字体
description: Crest 管理员维护系统参数、站点标识和字体资源的操作说明。
---

系统参数、站点设置和字体管理属于平台级配置。它们看起来是页面配置，实际会影响登录展示、资源渲染、导出效果和用户识别。

## 系统参数

入口：系统管理 → 系统参数。

![系统参数](/images/docs/crest/screenshots/system-parameter.png)

系统参数页控制平台级行为。图中每个参数都可能影响登录、查询、导出、分享或后台任务。修改前先记录原值和原因；修改后不要只看保存成功，要做一次登录、数据预览、仪表盘打开和导出检查。

截图中的系统参数页更适合由少数管理员维护，不建议开放给普通分析人员。参数修改属于平台变更，影响范围往往比页面上看到的更大。例如查询超时、导出限制、分享地址、任务配置这类参数，可能同时影响数据源预览、数据集缓存、仪表盘访问和导出中心。

管理员修改参数时要注意：

| 项目 | 建议 |
| --- | --- |
| 修改前 | 记录原值和修改原因 |
| 修改中 | 一次只改必要参数，避免多个问题叠加 |
| 修改后 | 验证登录、数据查询、导出和分享 |
| 出现异常 | 先回滚最近一次修改，再排查日志 |

## 站点设置

入口：系统管理 → 站点设置。

![站点设置](/images/docs/crest/screenshots/site-setting.png)

站点设置页用于调整系统名称、Logo、浏览器标题等品牌信息。公开交付前，管理员应在登录页、工作台、分享页和浏览器标签页分别检查一次，避免只改了后台配置但前台展示仍是旧名称。

站点设置常用于公开交付前统一品牌展示。

截图里的配置项一般不会影响数据计算，但会影响用户对系统的识别。公开项目尤其要注意名称、Logo 和浏览器标题一致：用户从登录页进入、从浏览器标签切换、从分享链接打开时，看到的应该是同一个产品或项目名称。

建议检查：

1. 登录页名称是否规范。
2. 浏览器标题是否符合项目要求。
3. Logo 是否清晰。
4. 页面底部或版权信息是否符合企业规范。
5. 分享页面是否也展示正确。

## 字体管理

入口：系统管理 → 字体管理。

![字体管理](/images/docs/crest/screenshots/font-management.png)

字体管理页影响仪表盘、大屏和导出效果。上传字体后，不代表所有页面自动使用该字体；需要在对应仪表盘或大屏编辑器中选择并保存。对外项目还要确认字体授权范围。

字体管理常用于数据大屏交付。上传字体前应确认授权范围，尤其是对外展示、商用项目和客户交付环境。

字体上传后建议用一张正式大屏做验证，而不是只看字体列表是否出现。因为字体真正生效的位置在编辑器和渲染结果中，特别是中文标题、数字指标、导出图片和投屏环境。对外项目要避免使用没有授权的商业字体。

操作流程：

<Steps>
  <Step>
    ### 准备字体
    确认字体文件来源、授权范围和格式。
  </Step>
  <Step>
    ### 上传字体
    在字体管理页面上传字体文件，名称保持简洁可识别。
  </Step>
  <Step>
    ### 应用到资源
    在仪表盘或大屏编辑器中选择字体，保存并预览。
  </Step>
  <Step>
    ### 导出验证
    使用图片、PDF 或预览功能检查字体是否生效。
  </Step>
</Steps>

## 管理建议

| 场景 | 建议 |
| --- | --- |
| 客户交付 | 统一 Logo、标题、字体和版权 |
| 多环境部署 | 测试、预发、生产保持配置差异可追踪 |
| 字体过多 | 清理未使用字体，避免编辑器选择混乱 |
| 参数变更频繁 | 建立配置变更审批和记录 |
| 页面展示异常 | 先排查站点配置、字体资源和浏览器缓存 |

## 交付前检查

1. 登录页截图是否符合项目形象。
2. 工作台名称和 Logo 是否正确。
3. 仪表盘和大屏是否使用统一字体。
4. 分享页面是否有正确品牌信息。
5. 导出文件是否能正确显示字体。

交付验收建议保留截图：登录页、工作台、一个仪表盘预览、一个大屏预览、一个导出文件。这样后续客户或业务方反馈品牌展示问题时，可以快速对比变更前后效果。

建议把站点配置变更写入交付清单，至少包含修改人、修改时间、旧值、新值和验收截图。这样后续升级、迁移或重装时，可以快速恢复一致的品牌展示。
