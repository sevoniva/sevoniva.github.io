---
title: 单机在线安装
description: 使用 installer/install.sh 在线安装 Crest，并理解关键配置和安装结果。
---

单机在线安装适合有网络访问能力的服务器。安装脚本会自动准备运行目录、配置文件、Docker Compose、系统服务和初始密码。

这种方式适合演示环境、小团队内部使用或生产规模较小的部署。正式使用前仍然要按生产标准准备域名、HTTPS、备份、密码和运维责任人。安装脚本能把服务跑起来，但不能替你完成安全和运维设计。

## 安装目录

在源码仓库中进入安装目录：

```bash
cd installer
```

主要文件：

| 文件 | 说明 |
| --- | --- |
| `install.sh` | 安装或升级入口 |
| `install.conf` | 安装配置模板 |
| `uninstall.sh` | 卸载脚本 |
| `crestctl` | 安装后的运维命令 |
| `crest/docker-compose.yml` | Docker Compose 模板 |
| `crest/templates/application.yml` | 后端配置模板 |

安装目录建议固定，不要每次升级换位置。后续备份、日志排查、服务重启和故障恢复都依赖这个目录。生产环境可以把数据目录放在独立磁盘，并把挂载关系写进运维文档。

## 修改安装配置

安装前先打开 `install.conf`，至少确认以下配置：

```bash
CREST_BASE=/opt
CREST_PORT=8100
CREST_CONTEXT_PATH=
CREST_LOGIN_TIMEOUT=960
CREST_APP_IMAGE=ghcr.io/sevoniva/crest:v1.5.0
CREST_MYSQL_IMAGE=mysql:8.4.5
CREST_ORIGIN_LIST=http://localhost:8100
CREST_CRYPTO_MODE=standard
```

生产环境建议显式配置：

```bash
CREST_PORT=8100
CREST_ORIGIN_LIST=https://bi.example.com
CREST_INITIAL_PASSWORD='替换为高强度初始密码'
CREST_DB_PASSWORD='替换为高强度数据库密码'
CREST_AES_KEY='32位加密Key'
CREST_AES_IV='16位加密IV'
CREST_CRYPTO_MODE=standard
```

如果留空，安装脚本会自动生成密码和加密参数，并在安装日志中输出。

生产环境不建议完全依赖临时输出。管理员应在安装前生成高强度密码和 AES 参数，并把它们保存到企业密码库或受控台账中。后续迁移、恢复和升级时，这些值必须能找到。

需要启用国密时，在首次安装前明确改为：

```bash
CREST_CRYPTO_MODE=sm-suite
CREST_SM4_KEY='0123456789abcdef'
```

`sm-suite` 模式用于需要 SM2、SM3、SM4 的环境。启用后，用户创建数据源、数据集、仪表盘和数据大屏的方式不变；运维侧需要把 `CREST_SM4_KEY` 和 AES Key、AES IV 一起纳入密钥管理，升级和恢复时不要更换。

## 执行安装

```bash
bash install.sh
```

安装脚本会依次完成：

<Steps>
  <Step>
    ### 读取配置
    加载 `install.conf`，确定安装目录、端口、镜像和密码策略。
  </Step>
  <Step>
    ### 初始化目录
    创建 `/opt/crest` 或你在 `CREST_BASE` 中指定的运行目录。
  </Step>
  <Step>
    ### 准备容器
    安装或复用 Docker 与 Docker Compose，拉取或加载应用和 MySQL 镜像。
  </Step>
  <Step>
    ### 生成运行配置
    写入数据库连接、加密模式、AES 参数、国密 SM4 Key、初始密码、允许来源等配置。
  </Step>
  <Step>
    ### 启动服务
    启动 MySQL 和 Crest，并注册系统服务。
  </Step>
  <Step>
    ### 输出登录信息
    输出访问地址、管理员账号和初始密码。
  </Step>
</Steps>

## 登录验证

浏览器访问安装输出的地址：

```text
http://服务器IP:8100
```

![登录页](/images/docs/crest/screenshots/login.png)

默认账号：

```text
用户名：admin
密码：安装完成后终端输出的初始密码
```

登录后进入工作台：

![工作台](/images/docs/crest/screenshots/workspace.png)

工作台出现后，安装只能算基本可用，还不能直接交付。继续进入系统管理检查站点名称、管理员密码、分享配置和导出中心；再创建一个测试数据源和数据集，确认数据库连接、文件目录和后台任务都能正常工作。

## 安装失败时先看哪里

| 现象 | 先检查 |
| --- | --- |
| Docker 安装失败 | 服务器是否能访问软件源，或是否需要使用离线包 |
| 镜像拉取失败 | 镜像仓库网络、代理、镜像名称 |
| 端口占用 | `ss -lntp | grep 8100` |
| 数据库启动失败 | MySQL 容器日志、磁盘空间、数据目录权限 |
| Crest 启动失败 | `docker compose logs -f crest` |

安装脚本日志应保留，便于后续定位初始密码、安装目录和错误原因。

## 安装后立即处理

<Steps>
  <Step>
    ### 修改管理员密码
    使用安装输出的初始密码登录后，立即进入个人菜单修改密码。
  </Step>
  <Step>
    ### 检查站点设置
    修改系统名称、Logo、浏览器标题等信息，确保交付页面不是临时名称。
  </Step>
  <Step>
    ### 创建验证资源
    创建一个测试数据源、数据集和仪表盘，确认完整链路可用。
  </Step>
  <Step>
    ### 配置备份
    至少备份数据库、运行配置、上传目录和安装脚本输出信息。
  </Step>
  <Step>
    ### 记录交付信息
    记录访问地址、安装目录、端口、版本、管理员负责人和维护命令。
  </Step>
</Steps>
