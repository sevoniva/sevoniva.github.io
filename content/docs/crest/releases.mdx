## 阅读建议

| 场景 | 建议 |
| --- | --- |
| 全新安装 | 优先选择最新正式版，按安装部署文档准备环境和密钥 |
| 离线部署 | 下载对应架构的离线包，并同时下载 checksums 文件做完整性校验 |
| 版本升级 | 先阅读当前版本和目标版本的发布说明，再按运维维护中的升级与回滚流程执行 |
| 问题排查 | 反馈问题时同时提供 Crest 版本号、部署方式、浏览器版本和相关日志 |

生产环境不建议只根据版本号直接替换镜像或离线包。升级前应完成备份、测试环境验证、维护窗口确认和回滚方案准备。
