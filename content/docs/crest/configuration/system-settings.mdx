系统参数、站点信息和字体配置影响 Crest 的展示、查询、导出和大屏效果。建议由平台管理员统一维护，不要让普通业务用户直接修改。

## 系统参数

入口：系统管理 → 系统参数。

![系统参数](/images/docs/crest/screenshots/system-parameter.png)

系统参数页面用于修改平台级配置。管理员进入页面后，先确认当前环境是测试、预发还是生产；生产环境修改前应记录原值。保存后至少验证登录、数据集预览、仪表盘打开、分享访问和导出任务。

系统参数页面通常用于维护平台级设置。修改前要先确认参数含义、默认值和影响范围。

截图中的参数列表不要当成普通表单随手改。建议管理员先搜索或定位目标参数，确认说明、当前值和默认值，再决定是否修改。参数变更后如果出现异常，优先回滚最近一次变更，而不是继续改更多参数。

| 配置方向 | 影响范围 | 建议 |
| --- | --- | --- |
| 基础参数 | 登录、展示、默认行为 | 改动前记录原值 |
| 查询参数 | 数据集预览、图表查询 | 与数据库能力和并发规模匹配 |
| 导出参数 | 导出中心、文件生成 | 结合文件大小和磁盘空间设置 |
| 任务参数 | 同步、缓存、后台任务 | 修改后观察日志和任务状态 |
| 安全参数 | 分享、访问、认证 | 与企业安全要求保持一致 |
| 账号安全 | 默认密码、密码策略、登录失败限制 | 与企业账号管理要求保持一致 |

<Callout type="info" title="参数修改建议">
  参数值不能只按“当前可运行”设置。正式环境要考虑数据增长、并发访问、导出峰值和后续运维成本。
</Callout>

## 账号安全参数

v1.5.5 起，系统参数中提供“账号安全”分组，用于统一维护本地账号的默认密码、密码策略和登录限制。该分组适合由平台管理员维护，普通用户不应具备修改权限。

常见参数：

| 参数 | 建议 |
| --- | --- |
| 默认密码 | 仅作为新建用户和重置密码后的临时密码，不应长期使用 |
| 密码策略 | 按企业规范设置长度、复杂度和校验规则 |
| 禁用初始密码 | 正式环境建议启用，避免用户长期使用初始密码 |
| 密码有效期 | 根据企业安全要求设置，过短会增加用户负担，过长会增加泄露风险 |
| 登录限制次数 | 防止暴力尝试，建议结合审计日志观察 |
| 登录限制时间 | 与限制次数配套，避免误伤正常用户 |

修改账号安全参数后，应至少验证三类场景：新建本地用户、重置用户密码、连续登录失败后的限制行为。若系统同时启用 SSO，还应确认 SSO 登录流程不受本地密码策略误影响。

<Callout type="warning" title="默认密码只能作为临时凭据">
  默认密码适合管理员创建账号或重置密码后临时交付给用户。生产环境不应把默认密码写入公开文档、截图、群公告或长期有效的项目交付材料。
</Callout>

## 站点设置

入口：系统管理 → 站点设置。

![站点设置](/images/docs/crest/screenshots/site-setting.png)

站点设置页面用于统一系统品牌。修改 Logo、系统名称和浏览器图标后，需要分别打开登录页、工作台、分享页和大屏预览页检查。配置页显示新值，并不等于所有访问入口都已刷新。

站点设置用于维护用户在浏览器和系统界面中看到的基本标识。

截图中的站点设置适合在上线前统一处理。名称、Logo、浏览器图标、版权信息要和项目交付口径一致。对外公开项目尤其要避免页面里出现测试名称、临时 Logo 或旧公司标识。

常见修改项：

| 项目 | 说明 |
| --- | --- |
| 系统名称 | 浏览器标题、页面标题和用户识别 |
| 站点 Logo | 登录页、侧边栏或顶部导航展示 |
| 浏览器图标 | 浏览器标签页图标 |
| 版权信息 | 按企业规范维护 |

修改后建议打开以下页面检查展示效果：

1. 登录页。
2. 工作台。
3. 仪表盘预览页。
4. 数据大屏预览页。
5. 分享链接页面。

## 字体管理

入口：系统管理 → 字体管理。

![字体管理](/images/docs/crest/screenshots/font-management.png)

字体管理页面用于上传和维护可视化字体。字体上传后，要到具体仪表盘或数据大屏中选择使用，再通过预览和导出验证。大屏交付项目尤其要检查目标展示机器是否能正常渲染。

字体会影响图表、仪表盘、大屏和导出效果。大屏项目如果使用了企业字体，必须在目标环境上传相同字体文件。

字体配置不是越多越好。字体过多会让编辑人员选择困难，也会增加导出和迁移时的维护成本。建议只保留项目真正使用的字体，并记录对应资源。

操作建议：

<Steps>
  <Step>
    ### 准备字体文件
    使用企业授权允许的字体文件，确认文件格式和授权范围。
  </Step>
  <Step>
    ### 上传字体
    在字体管理页面上传文件，并填写清晰的字体名称。
  </Step>
  <Step>
    ### 在大屏或仪表盘中使用
    编辑资源时选择已上传字体，保存后预览。
  </Step>
  <Step>
    ### 检查导出效果
    对使用该字体的仪表盘或大屏执行图片、PDF 或页面预览检查。
  </Step>
</Steps>

## 常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 修改站点名称后浏览器没变化 | 浏览器缓存或页面未刷新 | 强制刷新页面，必要时清理缓存 |
| 大屏字体丢失 | 目标环境未上传字体 | 在目标环境补充字体文件 |
| 导出字体与页面不一致 | 导出环境字体不可用 | 检查字体授权、文件状态和导出服务日志 |
| 参数改错导致异常 | 参数值不适合当前环境 | 回滚原值，重启相关服务并观察日志 |

## 变更记录建议

每次修改系统参数、站点或字体后，建议记录：

- 修改时间。
- 修改人。
- 修改前值和修改后值。
- 修改原因。
- 验证结果。
- 回滚方式。

可以把变更记录放在运维台账或项目交付文档中。记录要能支持两个问题：一是“为什么要改”，二是“如果出问题怎么恢复”。没有这两个答案，就不建议在生产环境直接修改。
