Crest 是面向私有化部署场景的开源 BI 平台，适用于企业内部数据分析、经营看板、项目交付和可视化大屏建设。平台覆盖数据源接入、数据集建模、图表制作、仪表盘搭建、数据大屏、分享发布、导出中心、数据血缘、用户权限和系统管理等完整流程。

本文档按实际使用场景组织。首次部署人员可从快速开始进入，运维人员可直接阅读安装部署和运维维护，业务用户可按用户指南完成数据分析页面，管理员可按管理员指南完成账号、权限、SSO 和审计治理。

![Crest 工作台](/images/docs/crest/screenshots/workspace.png)

## 产品能力

| 能力 | 说明 |
| --- | --- |
| 数据接入 | 支持数据库、Excel、API 等数据源，提供连接测试、表结构读取和数据预览 |
| 数据建模 | 通过数据集管理字段、指标、维度、权限、缓存和参数 |
| 可视化分析 | 支持图表、仪表盘和数据大屏，覆盖日常分析和展示大屏 |
| 发布协作 | 支持资源分享、导出中心、预览和外部访问治理 |
| 数据治理 | 提供数据血缘、权限控制、审计日志和平台级管理能力 |
| 私有化部署 | 支持单机、离线、Docker、Kubernetes 和外部 MySQL 部署方式 |
| 国密支持 | v1.5.0 支持 `standard` 和 `sm-suite` 两种加密模式，满足需要 SM2、SM3、SM4 的部署场景 |

## 推荐阅读路径

<Steps>
  <Step>
    ### 先部署
    如果你还没有可访问的 Crest 环境，请从“快速开始”进入，再按“安装部署”完成安装。
  </Step>
  <Step>
    ### 再配置
    生产环境部署完成后，先确认访问端口、系统地址、HTTPS、数据库连接、备份目录和管理员账号。
  </Step>
  <Step>
    ### 开始使用
    业务用户从工作台进入，按“数据源、数据集、图表、仪表盘或大屏”的顺序完成分析页面。
  </Step>
  <Step>
    ### 做好治理
    管理员应尽早配置组织、角色、资源权限、分享安全、SSO 和审计日志，避免后期权限边界不清。
  </Step>
</Steps>

## 版本说明

本文档面向 Crest v1.5.0。v1.5.0 支持默认 `standard` 加密模式，也支持可选的 `sm-suite` 国密模式。启用国密后，数据源、数据集、图表、仪表盘、数据大屏、分享、导出和数据血缘的用户操作流程保持一致，差异主要体现在部署参数、密钥保管和升级维护要求上。

不同部署方式、账号权限和现场配置会影响菜单展示、演示数据和默认参数。若页面中没有出现某个入口，优先检查当前账号角色、菜单权限、资源权限和部署版本。
